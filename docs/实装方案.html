01 为什么要做 / 现状问题

现有 mole-web 是 Flask + 原生 JS 的本地 Web 控制台，功能完整且已验证，但分发是硬伤。

现状痛点

  • 需用户自行装 Python 3.13 + pip install flask psutil
  • 需手动 python app.py 再开浏览器访问 127.0.0.1:5005
  • 端口、进程、启动全靠手动，普通用户无法使用

桌面版收益

  • 双击安装、双击运行，原生窗口，无浏览器
  • Python 运行时 + Flask + psutil 由 PyInstaller 全部打进 server.exe
  • 启动自动提权（管理员），优化/卸载全功能可用
  • 内置自动更新，后续版本一键升级

02 运行架构

Electron 主进程作为"宿主"：负责提权、找端口、拉起后端、健康探测、开窗、退出清理。渲染进程仍是原来的 6 标签页 UI。

NSIS 安装包Mole Setup 1.0.0.exe → 安装到 Program Files
▼
Electron 主进程 · main.js管理员守卫 → 空闲端口 → spawn server.exe → 轮询 /api/version → 开窗 → 退出 taskkill /t
▼
PyInstaller · server.exe内含 Python+Flask+psutil，绑定 127.0.0.1:{动态端口}
▼
BrowserWindow（渲染进程）loadURL(http://127.0.0.1:{port}) → core.js + 6 个视图
为什么用 PyInstaller 而非把后端用 Node 重写：mole-web 的 6 个命令逻辑（注册表枚举、psutil 指标、安全护栏、SSE 流）已写好并验证，PyInstaller 打包可零改写复用，风险最低、最快落地。

04 后端适配（仅 3 处）

把 mole-web/app.py 复制为 python/server.py，仅改这三处，其余 6 命令 API 原样保留。

#改动原因
1端口参数化
argparse --port，app.run(port=args.port)		Electron 传动态空闲端口，避免占用冲突
2资源路径
base = sys._MEIPASS or 脚本目录，Flask 的 template_folder / static_folder 指向它		打包后 templates/static 解压在临时目录，须用 _MEIPASS 定位
3去掉 Mole 仓库依赖
删 MOLE_ROOT，/api/version 返回内置 APP_VERSION		原来读 Mole 源码里的 VERSION 文件，用户机器上不存在
保持不变：UTF-8 stdout 包装、host=127.0.0.1（仅回环）、全部安全护栏（ProtectedPaths / 年龄阈值 / 「仅预览」默认勾选）、ShellExecuteW runas 卸载提权。

05 Electron 主进程要点

关键能力 main.js

  • 管理员守卫：net session 判断是否提权；否则 Start-Process -Verb RunAs 重启自身后退出
  • 空闲端口：net.createServer listen(0)
  • 拉后端：打包态 resources/server/server.exe，dev 态 python python/server.py
  • 健康探测：轮询 /api/version 200 再开窗，超时 30s
  • 退出清理：taskkill /pid /f /t 杀进程树
  • 单实例锁 + 启动 5s 后查更新

窗口 / 安全 BrowserWindow

  • 1400×900，min 1000×640，暗底 #0d1117
  • contextIsolation:true / nodeIntegration:false
  • preload 经 contextBridge 暴露最小 API
  • removeMenu()，ready-to-show 再显示（无白屏）

权限策略 管理员

启动即提权 → Optimize 的 Winsock/ARP/sfc、Uninstall 的机器级应用全部直接可用。代价：每次启动一次 UAC。

07 自动更新

机制 electron-updater

  • provider = github，启动 5s 后静默检查
  • autoDownload=false：发现新版先通知，用户确认再下
  • 下载进度 / 完成事件经 IPC 转发渲染进程
  • quitAndInstall 退出即装

前置条件 待用户提供

  • 一个用于发布的 GitHub 仓库（填入 package.json 的 publish.owner/repo）
  • 发布时设 GH_TOKEN 环境变量
  • 不发布也能本地出安装包，自动更新留作后续启用

08 验证清单（end-to-end）

  • dev 跑通 — npm start，主进程拉起本机 python，6 标签逐个验证（Clean 扫描 / Status 实时 / Analyze 钻取 / Optimize 列表 / Uninstall 清单）
  • 后端 exe — npm run build-backend；server.exe --port 5050 能独立起、浏览器能开
  • 出安装包 — npm run dist → dist/Mole Setup 1.0.0.exe
  • 装机验证 — 安装 → 启动弹 UAC → 6 标签全可用 → 测 Optimize 的 DNS 刷新成功 → 测 Uninstall 弹原生卸载器
  • 进程清理 — 关窗后任务管理器确认 server.exe 已被 taskkill /t，端口释放
  • 自动更新 —（可选）填好 repo 后 release，旧版能检测新版
  • 截图核验 — chrome-devtools MCP 连 dev 窗口，确认各页渲染（含已修复的 fmtRate 解绑 / 端口多监听坑）

09 已知坑（房型经验）

  • core.js helper 解绑调用：const fr=M.fmtRate 后 this 丢失，函数体内须用 MOLE. 而非 this.（已修，复制时保留）
  • Werkzeug 端口多监听：Windows 上同端口可能被多进程同时 LISTEN；Electron 用动态空闲端口规避
  • PyInstaller add-data 分隔符：Windows 用 ;；psutil 可能需 --hidden-import
  • onedir 产物较大：Flask+psutil 约 40–80MB，属正常
  • analyze 默认测整个用户目录较慢（AppData 151GB 单线程）；用户可输入子目录加速
